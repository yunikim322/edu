# edu/ 모바일 교육자료 디자인 가이드

실로암방문요양지원센터 직원 교육 포털(GitHub Pages)의 모바일 교육자료 HTML 제작 시 **반드시 지켜야 할 최소 기준**만 정리한 문서입니다. 이 기준만 지키면 디자인은 교육과정 특성에 맞게 자유롭게 구성합니다.

---

## 대상 사용자

- 60대 이상 포함 요양보호사
- 스마트폰으로 접속 (모바일 퍼스트)
- 긴 글은 읽지 않으므로 **핵심만 간결하게**

---

## 가독성 최소 기준

### 폰트
- **서체**: Noto Sans KR (Google Fonts import)

### 글자 크기 — 최솟값
- **본문 텍스트**: 19px 이상
- **보조 텍스트** (예시, 부연): 17px 이상

### 글자 색상 — 최소 명도
- **본문 텍스트**: #222 이하 (진하게)
- **보조 텍스트**: #555 이하
- **본문에 #333 이상 연한 색 사용 금지** — 모바일 야외 환경에서 가독성 저하

## 공통 레이아웃 구조

모든 교육자료 HTML은 아래 구조를 따릅니다:

```
스크롤 프로그레스 바
상단 바 (← 목록 | 페이지 제목) — sticky
히어로 (교육 제목 + 기관명)
목차 (챕터별 링크, 화살표 어포던스)
─── 콘텐츠 ───
  챕터 1
  챕터 2
  ...
  마무리
─── /콘텐츠 ───
푸터 (© 기관명)
플로팅 ↑ 버튼
```

---

## 지켜야 할 UI 규칙

1. **목차에 클릭 어포던스** — 화살표(›) 등으로 클릭 가능함을 표시
2. **scroll-margin-top 설정** — 목차 클릭 시 상단바가 콘텐츠를 가리지 않도록
3. **마무리 뱃지는 wrap 허용** — 하단 키워드 강조용 뱃지는 `flex-wrap: wrap` + `justify-content: center`로 자연스럽게 2줄 배치. nowrap + 가로 스크롤 금지 (강조 목적이라 모두 보여야 함)
4. **효과 태그 등 인라인 태그는 nowrap** — 챕터 내 효과 태그처럼 1줄에 들어가는 짧은 태그는 `flex-wrap: nowrap` + `overflow-x: auto`
5. **히어로 타이틀 줄바꿈 방지** — `word-break: keep-all`, 필요 시 `letter-spacing` 조정

---

## 콘텐츠 작성 원칙

1. **타이틀은 1줄로** — 줄바꿈으로 2줄 되면 가독성 떨어짐
2. **텍스트는 핵심만** — 자세한 설명은 동영상에서, 교육자료는 짧고 간결하게
3. **동일 카드 반복 지양** — 같은 형태가 연속되면 지루함. 카테고리 헤더, 컬러 바 등으로 변화
4. **강조 배너도 1줄** — 불필요한 2줄 금지
5. **뱃지 문구는 짧게** — 3~4글자 이내 권장

## PC 반응형 가이드

모든 교육자료 HTML은 **모바일 퍼스트 + PC 반응형**입니다. 모바일 CSS를 기본으로 두고, `@media` 쿼리로 PC 레이아웃을 덮어씁니다.

### 구조

```
<body>
  <div class="progress-bar">
  <div class="pc-wrapper">        ← PC에서 max-width + 중앙 정렬 + 그림자
    상단바 / 히어로 / 목차 / 콘텐츠 / 푸터
  </div>
  플로팅 ↑ 버튼
</body>
```

- `.pc-wrapper` — 모바일에서는 빈 규칙(투명), PC에서 max-width 1100~1200px + 중앙 정렬 + box-shadow
- `.progress-bar`와 `.fab-top`은 pc-wrapper **밖**에 배치 (fixed 요소)

### 브레이크포인트

| 구간 | 너비 | 특징 |
|------|------|------|
| 모바일 | ~767px | 기본 스타일 (변경 없음) |
| 태블릿/PC | 768px~ | 목차 가로화, 카드 2열 그리드, 3열 flex 등 |
| 넓은 PC | 1200px~ | max-width 확대, 3열 그리드, 패딩 확대 |

### PC에서 달라지는 레이아웃

1. **목차** — 세로 리스트 → 가로 탭 바 (`flex-direction: row`), `.toc-arrow` 숨김
2. **증상/학대유형/예방지침 리스트** — 1열 → 2~3열 그리드 (`grid-template-columns`)
3. **단계 카드 / 규칙 카드** — 세로 → 가로 3열 flex (`.stage-grid-pc`, `.rule-grid-pc` 등 래퍼 div 필요)
4. **체크 그룹** — 세로 → 가로 2열 flex (`.check-grid-pc` 래퍼)
5. **모든 카드** — hover 시 `translateY(-2~3px)` + box-shadow 확대
6. **챕터 헤더** — 하단 border 구분선 추가
7. **히어로** — 패딩 확대, 폰트 36~42px

### 래퍼 div 네이밍 규칙

PC에서 flex/grid 가로 배치가 필요한 카드 그룹은 래퍼 div로 감싼다:
- `.stage-grid-pc` — 단계 카드 (초기/중기/말기)
- `.rule-grid-pc` — 규칙 카드 (3권/3금/3행)
- `.check-grid-pc` — 체크 그룹 (확인항목/대처방법)
- 모바일에서는 빈 규칙으로 영향 없음

### 주의사항

- **모바일 CSS는 절대 건드리지 않는다** — PC 스타일은 반드시 `@media` 안에서만
- 래퍼 div는 모바일에서 빈 규칙(`{ }`)을 기본으로 선언해야 함
- 교육과정별 테마 색상(블루/오렌지/퍼플)에 맞게 hover 색상도 맞출 것
